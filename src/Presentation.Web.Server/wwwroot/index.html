bITdevKit GettingStarted API

Domain-Driven Design Example with Clean Architecture

Welcome

This is a demonstration API built with .NET 10 and the bITdevKit framework, showcasing a modular, Domain-Driven Design (DDD) application following Clean Architecture principles.

The API provides customer management capabilities with a full CRUD implementation, demonstrating best practices for enterprise application development.

Key Features

Modular Architecture

Vertical slice architecture with self-contained modules for better separation of concerns and maintainability.

Clean Architecture

Onion architecture with clear boundaries between Domain, Application, Infrastructure, and Presentation layers.

CQRS Pattern

Separate Command and Query handlers using the Requester/Notifier pattern for optimized read and write operations.

Domain Events

Event-driven architecture with domain events for loose coupling and reactive business logic.

Repository Pattern

Generic repositories with decorator behaviors for logging, auditing, and domain event publishing.

Result Pattern

Railway-oriented programming with Result types for explicit error handling without exceptions.
